Deployment scaffold is now available. It supports:

- remote AMD/ROCm preflight;

## 5. Observations

- The champion solution is CUDA/JAX-oriented via `jax[cuda12]`; do not assume it runs on AMD without a ROCm compatibility port.
- Sana's provided `environment_setup.sh` installs CUDA 12.8 PyTorch wheels and CUDA-oriented packages; do not run it as-is on AMD.
- NVIDIA render server and AMD training server should initially be treated as separate environments joined by synchronized code and serialized rollout data.
- AMD access is tunnel-only, so local SSH/rsync cannot be assumed. Deployment to AMD now uses a local tarball package and commands run inside the AMD VS Code Tunnel terminal.
- AMD training nodes are expected to be 8x MI325X under Slurm, with the reserved full-node allocation command recorded in docs.
